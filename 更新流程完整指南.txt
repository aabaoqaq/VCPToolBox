## �� 更新流程

### 方式1：使用自动更新脚本（推荐）

```
步骤1：双击 自动更新.bat
步骤2：查看更新内容
步骤3：按 Y 确认合并
步骤4：
  - 如果没有冲突 → 完成 ✅
  - 如果有冲突 → 继续下面的流程 ⚠️
```

### 方式2：手动更新（高级）

```powershell
# 第1步：切换到 custom 分支
git checkout custom

# 第2步：获取官方更新
git fetch upstream

# 第3步：查看官方更新了什么（可选）
git log custom..upstream/main --oneline -10

# 第4步：合并官方更新
git merge upstream/main

# 第5步：处理结果
# - 如果显示 "Already up to date" → 已经是最新 ✅
# - 如果显示 "Fast-forward" → 自动合并成功 ✅
# - 如果显示 "CONFLICT" → 有冲突，继续下面的流程 ⚠️
```

## �� 冲突识别

### 如何知道有冲突？

**脚本会显示：**
```
CONFLICT (content): Merge conflict in 文件名
Automatic merge failed; fix conflicts and then commit the result.
```

**或者在终端执行：**
```powershell
git status
```

**输出示例：**
```
On branch custom
You have unmerged paths.

Unmerged paths:
  (use "git add <file>..." to mark resolution)
        both modified:   Agent/ThemeMaidCoco.txt    ← 冲突文件

Changes to be committed:
        new file:   dailynote/可可的知识/CSS动画知识库.txt
        new file:   dailynote/可可的知识/示例主题.txt
```

### 识别冲突文件的符号

| 符号 | 说明 | 需要处理 |
|------|------|---------|
| **UU** | **未合并（冲突）** | ✅ **必须处理** |
| **both modified** | **双方都修改了** | ✅ **必须处理** |
| A | 新增文件 | ❌ 自动处理 |
| M | 修改文件 | ❌ 自动处理 |
| ?? | 未跟踪文件 | ❌ 不影响 |

## ��️ 冲突解决

### 第1步：列出所有冲突文件

```powershell
git status --short
```

**输出示例：**
```
UU Agent/ThemeMaidCoco.txt         ← 这是冲突文件
A  dailynote/可可的知识/CSS动画知识库.txt
A  dailynote/可可的知识/示例主题.txt
?? 自动更新.bat
```

**只需要处理 UU 开头的文件！**

---

### 第2步：查看具体冲突（可选）

#### 方式A：在终端查看

```powershell
git diff Agent/ThemeMaidCoco.txt
```

#### 方式B：在 VS Code 查看（推荐）

1. 在 VS Code 中打开冲突文件
2. 会看到高亮标记：

```
<<<<<<< HEAD (你的版本)
你的内容在这里
=======
官方的内容在这里
>>>>>>> upstream/main (官方版本)
```

**说明：**
- `<<<<<<< HEAD` 到 `=======` 之间：你的版本
- `=======` 到 `>>>>>>> upstream/main` 之间：官方版本

---

### 第3步：选择解决方式

你有 **3 种选择**：

#### 选择1：保留你的版本 ✅（推荐）

**适用场景：**
- 你修改过这个文件
- 你的配置很重要
- 不确定该用哪个

**命令：**
```powershell
git checkout --ours 文件路径
```

**例子：**
```powershell
git checkout --ours Agent/ThemeMaidCoco.txt
```

**解释：**
- `--ours` = 我们的（你的版本）
- 会完全使用你的版本，丢弃官方修改

---

#### 选择2：使用官方版本

**适用场景：**
- 你没改过这个文件
- 想要官方最新功能
- 确定官方版本更好

**命令：**
```powershell
git checkout --theirs 文件路径
```

**例子：**
```powershell
git checkout --theirs Agent/ThemeMaidCoco.txt
```

**解释：**
- `--theirs` = 他们的（官方版本）
- 会完全使用官方版本，丢弃你的修改

---

#### 选择3：手动合并（高级）

**适用场景：**
- 你想保留部分你的内容
- 同时也想要部分官方内容
- 你懂得如何编辑代码

**步骤：**

1. 在 VS Code 中打开冲突文件
2. 手动编辑，删除不要的部分
3. 删除 Git 标记：
   ```
   <<<<<<< HEAD
   =======
   >>>>>>> upstream/main
   ```
4. 保存文件

---

### 第4步：标记冲突已解决

**对每个冲突文件执行：**

```powershell
git add 文件路径
```

**例子：**
```powershell
# 单个文件
git add Agent/ThemeMaidCoco.txt

# 或者一次性添加所有已解决的冲突
git add -A
```

**解释：**
- `git add` 告诉 Git："这个文件的冲突我已经解决了"

---

### 第5步：完成合并

```powershell
git commit -m "合并官方更新，解决冲突"
```

**或者使用更详细的说明：**
```powershell
git commit -m "合并官方更新，保留我的 ThemeMaidCoco.txt 配置"
```

**解释：**
- `-m` = message（提交说明）
- 用中文描述你做了什么

---

### 第6步：验证合并成功

```powershell
git status
```

**应该显示：**
```
On branch custom
nothing to commit, working tree clean
```

**如果还有 "Unmerged paths"，说明还有冲突没解决，回到第1步！**

## �� 上传到 GitHub

### 为什么要上传？

- ✅ 备份你的代码到云端
- ✅ 在其他电脑可以同步
- ✅ 防止本地文件丢失

### 上传步骤

#### 第1步：推送到你的 Fork

```powershell
git push origin custom
```

**解释：**
- `origin` = 你的 GitHub 仓库
- `custom` = 分支名
- 把本地的 custom 分支推送到 GitHub

---

#### 第2步：处理推送结果

##### 情况A：成功 ✅

**显示：**
```
Enumerating objects: ...
Writing objects: 100% ...
To https://github.com/aabaoqaq/VCPToolBox.git
   abc1234..def5678  custom -> custom
```

**说明：** 已成功推送到 GitHub！

---

##### 情况B：需要登录 ��

**显示：**
```
Username for 'https://github.com':
```

**解决：**

1. **输入用户名：**
   ```
   aabaoqaq
   ```

2. **输入密码（Token）：**
 
   ⚠️ **不是你的 GitHub 登录密码！**
 
   需要用 **Personal Access Token**

**如何生成 Token：**

1. 打开：https://github.com/settings/tokens
2. 点击 **"Generate new token"** → **"Generate new token (classic)"**
3. **Note（备注）：** 写 "VCPToolBox"
4. **Expiration（有效期）：** 选 "No expiration"（永不过期）
5. **勾选权限：** 只需要勾选 **repo**（包含所有子选项）
6. 点击最下面的 **"Generate token"**
7. **复制生成的 Token**（以 `ghp_` 开头，只显示一次！）

**粘贴 Token：**
- 在命令行密码输入框粘贴（看不见字符是正常的）
- 按回车

---

##### 情况C：推送被拒绝 ⚠️

**显示：**
```
! [rejected]        custom -> custom (non-fast-forward)
error: failed to push some refs
```

**原因：** GitHub 上的代码比你本地的新（可能在其他电脑修改过）

**解决：**

```powershell
# 第1步：拉取 GitHub 上的最新代码
git pull origin custom

# 第2步：如果有冲突，解决冲突（参考上面的流程）

# 第3步：重新推送
git push origin custom
```

---

#### 第3步：验证上传成功

**在浏览器访问：**
```
https://github.com/aabaoqaq/VCPToolBox
```

**应该看到：**
- custom 分支有新的 commit
- 最新的提交时间是刚才
- 提交说明是你写的内容

## �� 常见问题

### Q1: 冲突太多，我想放弃合并

**命令：**
```powershell
git merge --abort
```

**说明：**
- 取消本次合并
- 回到合并前的状态
- 你的修改不会丢失

---

### Q2: 我想看看官方改了什么

**查看官方最新提交：**
```powershell
git log upstream/main --oneline -10
```

**查看具体文件的改动：**
```powershell
git diff custom..upstream/main -- Agent/ThemeMaidCoco.txt
```

---

### Q3: 我不小心选错了版本

**只要还没推送到 GitHub，就可以回退：**

```powershell
# 查看最近的提交
git log --oneline -5

# 回退到上一个提交
git reset --hard HEAD~1

# 重新解决冲突
```

---

### Q4: 冲突解决后，代码运行出错

**可能原因：**
- 保留你的版本，但官方有重要修复
- 建议重新合并，这次选择官方版本

**重新合并：**
```powershell
# 回退到冲突前
git reset --hard HEAD~1

# 重新解决冲突，这次选择官方版本
git checkout --theirs Agent/ThemeMaidCoco.txt
git add Agent/ThemeMaidCoco.txt
git commit -m "使用官方版本"
```

---

### Q5: 如何查看我改了哪些文件

```powershell
# 查看所有修改
git status

# 查看具体改了什么内容
git diff

# 查看某个文件的改动
git diff Agent/ThemeMaidCoco.txt
```

---

### Q6: 多个冲突文件怎么办

**一个一个解决：**

```powershell
# 查看所有冲突
git status --short

# 假设有 3 个冲突
git checkout --ours Agent/ThemeMaidCoco.txt
git add Agent/ThemeMaidCoco.txt

git checkout --theirs Plugin/SomePlugin/config.py
git add Plugin/SomePlugin/config.py

# 第3个手动合并（在 VS Code 中编辑）
# 编辑完后
git add Plugin/AnotherPlugin/settings.json

# 全部解决后提交
git commit -m "解决所有冲突"
```

---

### Q7: 我想同时保留两个版本

**方法：改文件名**

```powershell
# 保留你的版本为 _my
git show :2:Agent/ThemeMaidCoco.txt > Agent/ThemeMaidCoco_my.txt

# 保留官方版本为 _official
git show :3:Agent/ThemeMaidCoco.txt > Agent/ThemeMaidCoco_official.txt

# 解决冲突（随便选一个）
git checkout --ours Agent/ThemeMaidCoco.txt

# 添加所有文件
git add -A
git commit -m "保留两个版本"
```

## �� 完整流程总结

```
1. 双击 自动更新.bat（或手动 git merge upstream/main）
   ↓
2. 如果没有冲突
   └─→ 完成 ✅

3. 如果有冲突
   ↓
4. git status（查看冲突文件）
   ↓
5. 选择解决方式：
   ├─ git checkout --ours 文件（保留你的）
   ├─ git checkout --theirs 文件（使用官方）
   └─ 手动编辑（在 VS Code 中）
   ↓
6. git add 文件（标记已解决）
   ↓
7. git commit -m "解决冲突"（完成合并）
   ↓
8. git push origin custom（推送到 GitHub）
   ↓
9. 完成 ✅
```

## ⚠️ 重要提醒

1. **每次解决冲突前，先 `git status` 查看状态**
2. **不确定用哪个版本，优先用 `--ours`（保留你的）**
3. **解决完冲突一定要 `git push` 备份到 GitHub**
4. **遇到问题先 `git merge --abort` 取消，再重新来**
5. **所有操作都在 custom 分支进行，不要切换到 main**
